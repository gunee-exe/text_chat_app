# Chats — Flutter + Firebase Text Chat App

A minimal, distraction-free 1:1 and group text chat app built with Flutter and Firebase.

---

## Prerequisites

- Flutter SDK (latest stable)
- Firebase CLI: `npm install -g firebase-tools`
- FlutterFire CLI: `dart pub global activate flutterfire_cli`
- A Firebase project (create at [console.firebase.google.com](https://console.firebase.google.com))

---

## Setup

### 1. Enable email/password authentication (manual Firebase Console step)

1. Go to [Firebase Console](https://console.firebase.google.com) → your project
2. Navigate to **Authentication** → **Sign-in method**
3. Enable **Email/Password**
4. Save

### 2. Configure Firebase (required before running the app)

```bash
# Log in to Firebase
firebase login

# In the project root directory:
flutterfire configure
```

- Select your Firebase project
- Select your target platforms (Android, iOS, etc.)
- This generates `lib/firebase_options.dart` replacing the placeholder

### 3. Install dependencies

```bash
flutter pub get
```

### 4. Deploy Firestore security rules

```bash
firebase deploy --only firestore:rules
```

> Make sure `firestore.rules` is listed in your `firebase.json` under the `firestore` key:
> ```json
> {
>   "firestore": {
>     "rules": "firestore.rules"
>   }
> }
> ```
> Run `firebase init firestore` if you don't have a `firebase.json` yet.

### 5. Run the app

```bash
flutter run
```

## Firestore indexes

The following composite indexes are needed and will be prompted by Firebase on first run:

| Collection | Fields | Order |
|---|---|---|
| `chats` | `participants` (array), `lastMessageAt` | descending |
| `users` | `shortIdLower` (string) | — |

Deploy them via the Firebase Console link in the error message when first running, or add them manually in **Firestore → Indexes**.

## Known limitations (MVP scope)

- **No E2E encryption**: Messages are stored as plaintext in Firestore. This is flagged in `firestore_service.dart`.
- **No media sharing**: Text only. The mic icon in the chat list is a placeholder — tapping shows "coming soon".
- **No AI features**: No smart replies, summarization, or translation.
- **Group chat creation**: The data model supports groups, but a group creation UI is not included in MVP. See `// TODO` in `add_friend_sheet.dart`.
- **Push notification delivery**: FCM token registration is implemented; actual server-side notification sending requires a Cloud Function or an FCM-capable backend (not included in this MVP).
